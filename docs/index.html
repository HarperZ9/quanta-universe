What is QuantaLang?

QuantaLang is a systems programming language that compiles to native binaries via C. The compiler is written in Rust (80K lines, 650 tests, 8 backends) and includes a self-hosted compiler written in QuantaLang itself (2,490 lines).

The project includes 62 CLI tools — from wc and grep to a SQL database engine with transactions — all compiled from QuantaLang source.

Features

  • Structs, enums, pattern matching, traits, generics
  • Closures, iterators (.map/.filter/.fold)
  • String operations, Vec, HashMap
  • File I/O, directory traversal, TCP sockets
  • include!() preprocessor for multi-file projects
  • 8 backends: C (primary), LLVM, WASM, SPIR-V, HLSL, GLSL, ARM64, x86-64

Program Suite (62 programs)

Self-Hosting Compiler Chain

source.quanta → qtok → qparse → qcheck → qcodegen → valid C code

The self-hosted compiler (qc, 1,898 lines) combines all four stages into a single binary. The qcodegen component (2,490 lines) handles functions, structs, impl methods, closures, Vec, include!(), and 30+ builtins. It generates valid C for all 62 programs and can compile itself.

Architecture

The compiler pipeline: preprocessor → lexer → parser → type checker → MIR lowerer → backend.

MIR (Mid-level IR) uses SSA-style basic blocks. The type system uses bidirectional inference with Hindley-Milner unification. See DESIGN.md for the full architecture documentation.
