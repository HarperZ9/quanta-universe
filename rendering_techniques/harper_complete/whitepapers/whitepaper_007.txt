Coherent Path Space Filtering: Denoising in Path Space for Edge-Preserving Global Illumination

Harper Research Division

Harper Engine Technical Whitepaper HE-WP-007

Abstract

We present Coherent Path Space Filtering (CPSF), a novel denoising approach operating in path space rather than screen space. By grouping paths by vertex chain similarity, CPSF eliminates edge bleeding artifacts inherent to image-space filters while achieving superior noise reduction.

1. Introduction

Screen-space denoising has become standard practice for real-time ray tracing. Filters like SVGF and A-SVGF exploit spatial coherence to reduce noise from low sample counts. However, screen-space approaches suffer fundamental limitations: they cannot distinguish paths that happen to land on adjacent pixels from paths that share geometric structure.

This leads to edge bleeding—light incorrectly spreading across geometric discontinuities. At object silhouettes, shadow boundaries, and material transitions, screen-space filters either produce bleeding artifacts (if too aggressive) or fail to reduce noise (if too conservative).

We propose Coherent Path Space Filtering (CPSF), moving the filtering operation from screen space to path space. Rather than grouping pixels by screen location, we group paths by their vertex chain similarity. Paths bouncing off similar geometry share a filter kernel regardless of final screen position.

This simple change yields dramatic improvements: edge bleeding is eliminated because paths crossing discontinuities have low similarity; noise reduction improves because genuinely similar paths (same surface, different sample) are correctly identified for averaging.

2. Path Space Representation

2.1 Path Encoding

A path P = {v₀, v₁, ..., vₖ} consists of vertices vᵢ, each storing:
- Position pᵢ ∈ ℝ³
- Normal nᵢ ∈ S²
- Material ID mᵢ
- BSDF type bᵢ (diffuse/glossy/specular)

We encode each vertex as a feature vector:

fᵢ = [pᵢ, nᵢ, one_hot(mᵢ), one_hot(bᵢ)]

The full path encoding concatenates vertex features with relative transformations:

F(P) = [f₀, f₁ - f₀, f₂ - f₁, ..., fₖ - fₖ₋₁]

2.2 Path Similarity Metric

Similarity between paths P₁ and P₂:

sim(P₁, P₂) = Πᵢ sim_vertex(v₁ᵢ, v₂ᵢ)

where vertex similarity combines spatial and angular terms:

sim_vertex(v₁, v₂) = exp(-||p₁ - p₂||²/σ_p²) · max(0, n₁·n₂)² · δ(m₁, m₂)

The Kronecker delta δ ensures material boundaries are respected.

2.3 Path Clustering

We cluster paths into coherent groups using locality-sensitive hashing:

hash(P) = sign(F(P) · R)

where R is a random projection matrix. Paths with identical hashes are candidates for the same filter kernel. Multiple hash functions reduce false negatives.

3. Path Space Filter

3.1 Filter Formulation

For a path P with radiance contribution L(P), the filtered estimate:

L̂(P) = Σᵢ w(P, Pᵢ) · L(Pᵢ) / Σᵢ w(P, Pᵢ)

where the sum ranges over paths in P's cluster and nearby clusters.

Weights combine path similarity with confidence:

w(P, Pᵢ) = sim(P, Pᵢ) · confidence(Pᵢ)

where confidence is inverse variance estimated from temporal history.

3.2 Kernel Adaptation

The filter kernel σ_p (spatial scale) adapts to local path density:

σ_p = σ_base · (target_samples / local_sample_count)^(1/3)

Regions with few samples (complex visibility) use wider kernels; well-sampled regions use tighter kernels for sharpness.

3.3 Temporal Integration

Path space filtering combines with temporal accumulation:

1. Filter current frame paths → filtered radiance
2. Reproject history using screen-space motion vectors
3. Validate history using path similarity to current frame
4. Blend filtered current with validated history

4. Implementation

4.1 Data Structures

struct PathVertex {
    float3 position;
    half3  normal;
    uint   material_id;
    uint   bsdf_type;
};

struct Path {
    PathVertex vertices[MAX_BOUNCES];
    uint       vertex_count;
    float3     contribution;
    uint       hash;
};

4.2 GPU Pipeline

Pass 1: Path Hashing
- Compute LSH hash for each path
- Parallel prefix sum for cluster offsets

Pass 2: Cluster Building  
- Scatter paths to cluster storage
- Sort within clusters by secondary hash

Pass 3: Filtering
- For each path: gather similar paths from cluster
- Compute weighted average
- Write filtered contribution to output

4.3 Memory and Performance

- Path storage: 64 bytes × samples × bounces ≈ 48MB at 1080p
- Cluster index: 4MB
- Filter pass: 1.8ms at 1080p, 1 spp

5. Results

5.1 Edge Bleeding Comparison

We measure light leaking across edges via ground-truth mask comparison:

Method              | Edge Error | Interior Error
--------------------|------------|---------------
SVGF                | 0.089      | 0.023
A-SVGF              | 0.054      | 0.021
NRD ReBLUR          | 0.041      | 0.019
CPSF (ours)         | 0.012      | 0.020

CPSF reduces edge bleeding by 3-7× versus screen-space alternatives.

5.2 Overall Quality

PSNR at 1 spp (higher is better):

Scene          | SVGF  | A-SVGF | NRD   | CPSF
---------------|-------|--------|-------|------
Sponza GI      | 28.4  | 29.1   | 30.2  | 31.4
Bistro Interior| 27.8  | 28.6   | 29.5  | 30.8
Shadow edges   | 26.2  | 27.4   | 28.1  | 30.9

5.3 Performance

At 1080p, 1 spp:
- SVGF: 2.1ms
- A-SVGF: 2.8ms
- NRD: 2.3ms
- CPSF: 3.2ms

CPSF is ~50% more expensive but provides substantially higher quality at edges.

6. Conclusion

Coherent Path Space Filtering demonstrates that denoising in path space rather than screen space fundamentally improves edge preservation while maintaining competitive noise reduction. The approach eliminates edge bleeding artifacts that plague all screen-space denoisers.

The technique is compatible with any path tracing renderer and combines naturally with temporal accumulation. Future work includes learned path similarity metrics, extension to path guiding (similar paths suggest similar importance), and hybrid approaches combining path-space and screen-space filtering for optimal quality-performance tradeoff.
