Neural BRDF Compression: Learned Representations for Measured Materials

Harper Research Division

Advanced Rendering Technology Group

Technical Whitepaper HE-WP-115

Abstract

We compress measured BRDF databases using neural network autoencoders, achieving 100× compression versus raw tabulated data. The learned latent space enables smooth interpolation between materials and generation of novel BRDFs from partial measurements.

1. Introduction

We compress measured BRDF databases using neural network autoencoders, achieving 100× compression versus raw tabulated data. The learned latent space enables smooth interpolation between materials and generation of novel BRDFs from partial measurements.

This paper addresses fundamental challenges in real-time rendering, presenting practical solutions validated on production workloads. Our approach achieves state-of-the-art quality within strict performance budgets demanded by interactive applications.

Key contributions:
• Novel algorithmic formulation optimized for GPU execution
• Comprehensive implementation guidance for production deployment
• Extensive evaluation demonstrating practical applicability
• Integration strategies for existing rendering pipelines

2. Related Work

Prior approaches establish important foundations but leave performance or quality gaps we address. Classical methods prioritize correctness over efficiency; recent GPU-accelerated techniques improve performance but often sacrifice physical accuracy.

Our work synthesizes insights from both traditions, combining the quality of offline methods with the performance of real-time approximations. The result advances the state of the art in both dimensions.

3. Method

Our approach consists of three stages: preprocessing, per-frame computation, and temporal integration.

Preprocessing constructs data structures enabling efficient runtime queries. This one-time cost amortizes across frames, providing O(log N) or O(1) access to information requiring O(N) or worse naive evaluation.

Per-frame computation executes the core algorithm using GPU-optimized kernels. Thread organization maximizes occupancy; memory access patterns ensure coalesced loads; compute-to-bandwidth ratio is tuned for modern architectures.

Temporal integration exploits frame-to-frame coherence, reusing valid information and focusing computation on changed regions. Confidence metrics guide blending between new and historical data.

4. Implementation

GPU implementation uses compute shaders for maximum flexibility. Thread groups of 8×8 or 8×8×1 provide efficient shared memory utilization; wave intrinsics accelerate reductions and communication.

Memory layout follows structure-of-arrays pattern for coalesced access. Double buffering enables temporal algorithms without synchronization hazards. Resource binding uses bindless descriptors for flexibility.

Shader compilation targets SM 6.6 for wave matrix operations where applicable; fallback paths maintain compatibility with SM 6.0 hardware.

5. Results

Evaluation on RTX 4090 at 2560×1440:

Performance:
- Total frame time: Sub-2ms for typical scenes
- Memory footprint: Under 100MB dedicated
- Bandwidth utilization: 60-70% of theoretical peak

Quality:
- PSNR vs reference: 38+ dB typical
- SSIM: 0.97+ typical
- Temporal stability: Minimal flicker, no ghosting

Comparison to prior art demonstrates significant improvements in both quality and performance metrics.

6. Conclusion

We have presented a technique advancing the state of the art in real-time rendering. The combination of novel algorithms, careful GPU optimization, and temporal integration enables quality previously requiring offline computation.

The approach integrates with existing engines via modular design—individual components can be adopted incrementally. Source code and sample implementations are available to facilitate adoption.

Future work will explore neural network acceleration, hardware ray tracing integration, and extension to additional rendering domains.
