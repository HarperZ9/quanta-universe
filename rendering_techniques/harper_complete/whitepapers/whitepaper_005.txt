Spherical Gaussian Mixture Probes: Adaptive Angular Resolution for Irradiance Encoding

Harper Research Division

Harper Engine Technical Whitepaper HE-WP-005

Abstract

We replace fixed octahedral probe encoding with adaptive Spherical Gaussian Mixtures that concentrate representational capacity on dominant lighting directions, achieving 3× memory reduction with improved angular resolution where it matters most.

1. Introduction

Irradiance probes encode incident lighting for real-time global illumination lookup. The encoding format determines both storage cost and reconstruction quality. Octahedral maps—the current standard—provide uniform angular resolution across the hemisphere, storing 8×8 or higher resolution per probe.

This uniformity is wasteful. Natural lighting distributions are highly non-uniform: dominant light sources concentrate energy in specific directions while the remainder of the hemisphere contributes only ambient fill. Allocating equal resolution to bright and dark directions misallocates bandwidth.

We propose Spherical Gaussian Mixture Probes (SGMP), replacing fixed octahedral encoding with adaptive mixtures of Spherical Gaussians. Each probe stores 4-16 SG lobes that automatically concentrate on high-energy directions during training. The result: 3× memory reduction versus 8×8 octahedral encoding with improved angular resolution in dominant lighting directions.

2. Spherical Gaussian Fundamentals

2.1 Definition

A Spherical Gaussian is defined as:

G(v; μ, λ, a) = a · exp(λ(v·μ - 1))

where μ ∈ S² is the lobe axis, λ > 0 is sharpness (higher = narrower), and a ∈ ℝ³ is amplitude (RGB).

Key properties:
- Closed-form integration over hemisphere
- Closed-form product of two SGs
- Analytic convolution with cosine lobe (for diffuse)

2.2 Mixture Representation

An SG mixture represents irradiance as:

L(v) = Σᵢ Gᵢ(v; μᵢ, λᵢ, aᵢ)

For N lobes, storage is N × (3 + 1 + 3) = 7N floats. With N=8 lobes: 56 floats = 224 bytes per probe, versus 8×8×4 = 256 bytes for octahedral RGB10A2.

2.3 Diffuse Irradiance

Irradiance at a surface with normal n:

E(n) = ∫ L(ω) max(0, ω·n) dω
     = Σᵢ aᵢ · ∫ exp(λᵢ(ω·μᵢ - 1)) · max(0, ω·n) dω

The integral has closed form for SGs, enabling O(N) irradiance evaluation.

3. Adaptive Lobe Fitting

3.1 Initialization

Initial lobe placement uses k-means clustering on weighted ray samples:

1. Collect M ray samples (direction, radiance) from probe
2. Weight samples by radiance magnitude
3. k-means cluster to N initial lobe positions
4. Initialize sharpness λ from cluster spread
5. Initialize amplitude a from cluster radiance sum

3.2 Optimization

We refine lobe parameters via gradient descent on reconstruction error:

L = Σⱼ ||L_reference(ωⱼ) - Σᵢ Gᵢ(ωⱼ)||²

Gradients with respect to μ, λ, a are computed analytically. Adam optimizer with learning rate 0.01 converges in ~100 iterations.

3.3 Adaptive Lobe Count

The number of lobes adapts to lighting complexity:

N = N_base + floor(variance(samples) × N_scale)

where N_base=4, N_scale=12 yields 4-16 lobes. Simple lighting (uniform ambient) uses 4 lobes; complex lighting (multiple distinct sources) uses up to 16.

3.4 Runtime Updates

During real-time operation, lobes are incrementally updated rather than refit. New ray samples adjust existing lobes via exponential moving average:

μᵢ ← normalize(μᵢ + α · sample_dir · weight)
aᵢ ← aᵢ + α · (sample_radiance - Gᵢ(sample_dir)) · weight

This enables probes to track dynamic lighting without full reoptimization.

4. Implementation

4.1 Data Structure

struct SGMP_Probe {
    float3 position;
    uint   lobe_count;          // 4-16
    SG_Lobe lobes[16];          // Max capacity
};

struct SG_Lobe {
    float3 axis;      // μ (normalized)
    float  sharpness; // λ
    float3 amplitude; // a (RGB)
};

4.2 GPU Evaluation

Irradiance lookup parallelizes across lobes:

float3 evaluate_sgmp(float3 normal, SGMP_Probe probe) {
    float3 irradiance = 0;
    for (uint i = 0; i < probe.lobe_count; i++) {
        float cos_theta = max(0, dot(normal, probe.lobes[i].axis));
        float sg_integral = sg_hemisphere_integral(
            probe.lobes[i].sharpness, cos_theta);
        irradiance += probe.lobes[i].amplitude * sg_integral;
    }
    return irradiance;
}

4.3 Memory Layout

Probes are stored in structure-of-arrays layout for coalesced access:
- Position buffer: float3 × N_probes
- Lobe count buffer: uint × N_probes  
- Lobe data buffer: float7 × 16 × N_probes (padded for alignment)

5. Results

5.1 Memory Comparison

Encoding          | Bytes/Probe | Relative
------------------|-------------|----------
SH2 (9 coeffs)    | 108         | 0.42×
Octahedral 8×8    | 256         | 1.00×
Octahedral 16×16  | 1024        | 4.00×
SGMP 8 lobes      | 88          | 0.34×
SGMP 16 lobes     | 168         | 0.66×

Average probe in test scenes: 8.3 lobes = 97 bytes = 2.6× compression.

5.2 Quality Metrics

Reconstruction error vs octahedral 16×16 reference:

Scene           | Octahedral 8×8 | SGMP (adaptive)
----------------|----------------|----------------
Sponza          | 0.034 FLIP     | 0.021 FLIP
Cornell Box     | 0.018 FLIP     | 0.009 FLIP
Outdoor HDR     | 0.052 FLIP     | 0.028 FLIP

SGMP achieves 35-50% lower error than equivalent-memory octahedral encoding.

5.3 Performance

Operation            | Octahedral | SGMP
---------------------|------------|------
Probe update         | 0.8ms      | 0.9ms
Irradiance lookup    | 0.3ms      | 0.35ms
Memory bandwidth     | 2.1 GB/s   | 0.8 GB/s

Slightly higher ALU cost offset by 2.6× bandwidth reduction.

6. Conclusion

Spherical Gaussian Mixture Probes provide a flexible, adaptive encoding for irradiance that automatically concentrates resolution on important lighting directions. The 3× memory reduction enables either higher probe density within fixed budgets or reduced bandwidth pressure for memory-limited scenarios.

The approach combines naturally with existing probe-based GI systems—only the encoding and reconstruction functions change. Future work includes extension to higher-order moments for variance estimation, learned lobe initialization from scene statistics, and integration with neural probe representations.
