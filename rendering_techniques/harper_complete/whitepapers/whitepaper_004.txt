Reservoir-Based Indirect Illumination: Extending ReSTIR to Multi-Bounce Global Illumination

Harper Research Division

Harper Engine Technical Whitepaper HE-WP-004

Abstract

We extend Reservoir-based Spatio-Temporal Importance Resampling (ReSTIR) from direct lighting to multi-bounce indirect illumination, achieving 9× variance reduction at 1 sample per pixel through world-space reservoir grids and path-aware resampling.

1. Introduction

ReSTIR revolutionized real-time direct lighting by demonstrating that reservoir-based resampling could achieve orders-of-magnitude variance reduction within fixed sample budgets. The algorithm's power derives from reusing samples across pixels and frames, effectively amortizing the cost of importance sampling.

Extending ReSTIR to indirect illumination presents fundamental challenges. Direct lighting involves sampling a known light distribution. Indirect lighting requires sampling the incident radiance field—itself the unknown we're computing. The recursive nature breaks the assumptions underlying standard ReSTIR.

We present Reservoir-Based Indirect Illumination (RBII), adapting ReSTIR principles for multi-bounce GI through three key innovations:

1. World-space reservoir grids storing incident radiance samples independently of view
2. Path-aware resampling weights accounting for BSDF and geometric factors along paths
3. Temporal persistence enabling sample reuse across hundreds of frames

RBII achieves 9.3× mean squared error reduction versus uniform sampling at 1 spp, enabling production-quality indirect illumination within strict frame budgets.

2. Background: ReSTIR for Direct Lighting

2.1 Reservoir Sampling

A reservoir maintains a weighted sample from a stream. Given samples xᵢ with weights wᵢ, the reservoir stores one sample such that P(stored = xᵢ) = wᵢ / Σⱼwⱼ. Crucially, this requires O(1) memory regardless of stream length.

The streaming update rule:

reservoir.w_sum += w_new
if random() < w_new / reservoir.w_sum:
    reservoir.sample = x_new

2.2 Resampled Importance Sampling (RIS)

RIS draws M candidates from a source distribution p(x) and selects one proportional to target distribution q(x):

weight(xᵢ) = q(xᵢ) / p(xᵢ)

The selected sample has effective PDF approaching q(x) as M increases, enabling importance sampling from complex distributions using simple candidate generation.

2.3 Spatial and Temporal Reuse

ReSTIR's key insight: neighboring pixels likely have similar optimal samples. By sharing reservoirs spatially (within constraints preserving unbiasedness) and temporally (across frames), effective sample counts increase dramatically without additional rays.

3. Extending to Indirect Illumination

3.1 World-Space Reservoir Grid

Unlike direct lighting where reservoirs are per-pixel, indirect illumination benefits from view-independent caching. We maintain a 3D grid of reservoirs in world space:

struct IndirectReservoir {
    float3 sample_direction;  // Incident direction
    float3 sample_radiance;   // Incoming radiance
    float  sample_distance;   // Path length
    float  w_sum;             // Cumulative weight
    uint   M;                 // Sample count
    uint   age;               // Frames since creation
};

Grid resolution adapts to local geometric density—finer near surfaces, coarser in open space.

3.2 Path-Aware Target Distribution

For indirect lighting at point p with normal n, the target PDF should be proportional to:

q(ω) = Lᵢ(p, ω) · f(ω, ωₒ) · cos(θ)

where Lᵢ is incident radiance (unknown), f is the BSDF, and cos(θ) is the foreshortening term.

We approximate Lᵢ using the reservoir's stored radiance estimate, updating both the sample and radiance estimate iteratively. This bootstrapping converges because temporal accumulation provides increasingly accurate Lᵢ estimates.

3.3 Resampling Weights

When combining reservoirs (spatial/temporal reuse), weights must account for the full path contribution:

w_combined = w_reservoir · (q_target / q_reservoir) · jacobian

The Jacobian term corrects for the geometric relationship between the original and reuse locations, ensuring unbiased combination.

3.4 Temporal Reuse with Path Validation

Temporal reuse requires validating that the stored sample remains valid:

1. Visibility: Re-trace shadow ray to confirm unoccluded
2. Relevance: Verify sample direction within BSDF lobe
3. Freshness: Decay weight based on age

Samples failing validation are discarded; their reservoirs receive reduced temporal weight.

4. Implementation

4.1 Grid Structure

The reservoir grid uses cascaded resolution:
- Cascade 0: 1m cells, 128³ grid (2M reservoirs)
- Cascade 1: 4m cells, 64³ grid (262K reservoirs)
- Cascade 2: 16m cells, 32³ grid (32K reservoirs)

Total memory: ~96MB for reservoir data.

4.2 Update Pipeline

Per frame:
1. Generate candidate samples via cosine-weighted hemisphere sampling
2. Trace rays, evaluate incident radiance
3. Update world-space reservoirs with new samples
4. During shading: query nearby reservoirs, combine with path-aware weights
5. Apply spatial reuse (3×3 neighborhood)
6. Output: resampled indirect illumination estimate

4.3 Optimizations

- Reservoir queries use cached grid lookups (0.02ms)
- Spatial reuse limited to 4 neighbors for performance
- Temporal decay prevents unbounded reservoir age
- Stratified candidate generation reduces correlation

5. Results

5.1 Variance Reduction

MSE relative to reference (lower is better):

Method                  | 1 spp MSE | 4 spp MSE
------------------------|-----------|----------
Uniform sampling        | 1.000     | 0.250
Cosine-weighted         | 0.612     | 0.153
RBII (ours)            | 0.107     | 0.031
RBII + spatial reuse   | 0.042     | 0.012

RBII achieves 9.3× variance reduction at 1 spp, 8.1× at 4 spp.

5.2 Performance

Component          | Time (ms)
-------------------|----------
Candidate generation | 0.1
Ray tracing        | 0.8
Reservoir update   | 0.2
Spatial reuse      | 0.15
Total              | 1.25

5.3 Convergence

RBII converges to reference quality 4× faster than standard path tracing due to sample reuse. For animated sequences, the persistent reservoir grid provides immediate high-quality estimates for newly visible regions.

6. Conclusion

Reservoir-Based Indirect Illumination extends the ReSTIR framework to multi-bounce global illumination, achieving nearly an order of magnitude variance reduction within fixed sample budgets. The world-space reservoir grid provides view-independent caching, while path-aware resampling weights ensure correct importance sampling.

The technique integrates with existing deferred rendering pipelines and combines naturally with temporal anti-aliasing. Future work includes extension to specular paths, integration with neural radiance caching, and exploration of learned resampling strategies.
