Persistent Thread Pools for GPU Compute: Work-Stealing Architecture for Dynamic Workloads

Harper Research Division

Advanced Rendering Technology Group

Technical Whitepaper HE-WP-053

Abstract

We present a persistent thread pool architecture for GPU compute that eliminates kernel launch overhead and enables dynamic load balancing through work stealing. Unlike traditional dispatch-per-task models, our approach launches long-running thread groups that continuously pull work from global queues. This achieves 3.2× throughput improvement for fine-grained tasks and enables adaptive subdivision, progressive refinement, and producer-consumer patterns previously impractical on GPUs.

1. Introduction

Traditional GPU programming follows a dispatch-per-task model: CPU submits kernel, GPU executes, CPU submits next kernel. Each dispatch incurs 5-50μs overhead from command buffer submission, pipeline flush, and scheduler wake-up. For fine-grained workloads (1000s of small tasks), this overhead dominates execution time.

More fundamentally, the dispatch model precludes dynamic work generation. A kernel cannot spawn additional work without completing, returning to CPU, and triggering a new dispatch. This blocks algorithms requiring:

- Adaptive subdivision (refine until error threshold)
- Producer-consumer pipelines (stage N feeds stage N+1)
- Work stealing (idle threads grab work from busy threads)

We introduce Persistent Thread Pools (PTP), an architecture where thread groups run indefinitely, pulling work from global queues until explicitly terminated. Work producers (CPU or other GPU threads) enqueue tasks; consumers dequeue and execute.

Key contributions:
1. Lock-free work queue design for GPU memory hierarchy
2. Work stealing protocol balancing load across SMs
3. Termination detection without global synchronization
4. Integration patterns for common rendering workloads

2. Architecture Overview

2.1 System Components

PTP consists of:
1. Work Queues: Per-SM local queues + global overflow queue
2. Worker Threads: Persistent thread groups executing tasks
3. Task Descriptors: Encoded work items with function pointers
4. Termination Protocol: Distributed quiescence detection

2.2 Work Queue Hierarchy

// Per-SM queue (fast path)
struct LocalQueue {
    uint head;           // Consumer reads from head
    uint tail;           // Producer writes to tail
    TaskDesc tasks[LOCAL_QUEUE_SIZE];
};

// Global queue (overflow + inter-SM work)
struct GlobalQueue {
    uint head;
    uint tail;
    TaskDesc tasks[GLOBAL_QUEUE_SIZE];
};

Producers first attempt local enqueue; on local queue full, overflow to global. Consumers first dequeue local; on local empty, steal from global or other SMs.

2.3 Task Descriptor

struct TaskDesc {
    uint functionId;     // Index into function table
    uint dataOffset;     // Offset into argument buffer
    uint dataSize;       // Argument byte count
    uint priority;       // For ordered execution
};

Function dispatch uses indirect call via function pointer table:

typedef void (*TaskFunc)(uint dataOffset, uint dataSize);
__device__ TaskFunc g_FunctionTable[MAX_FUNCTIONS];

void ExecuteTask(TaskDesc task) {
    g_FunctionTable[task.functionId](task.dataOffset, task.dataSize);
}

2.4 Worker Thread Structure

[numthreads(256, 1, 1)]
void PersistentWorker(uint tid : SV_DispatchThreadID) {
    uint smId = tid / 256;  // Which SM this thread belongs to
    LocalQueue* local = &g_LocalQueues[smId];
    
    while (true) {
        TaskDesc task;
        
        // Try local queue first
        if (TryDequeueLocal(local, task)) {
            ExecuteTask(task);
            continue;
        }
        
        // Try global queue
        if (TryDequeueGlobal(task)) {
            ExecuteTask(task);
            continue;
        }
        
        // Work stealing from other SMs
        if (TrySteal(smId, task)) {
            ExecuteTask(task);
            continue;
        }
        
        // No work available - check termination
        if (ShouldTerminate()) {
            return;
        }
        
        // Spin wait (with backoff)
        SpinWait();
    }
}

3. Lock-Free Queue Implementation

3.1 Local Queue (SPSC)

Each SM has a single-producer single-consumer queue. Producer is the owning thread group; consumer is the worker thread.

bool TryEnqueueLocal(LocalQueue* q, TaskDesc task) {
    uint tail = atomicLoad(&q->tail);
    uint head = atomicLoad(&q->head);
    
    if (tail - head >= LOCAL_QUEUE_SIZE) {
        return false;  // Queue full
    }
    
    q->tasks[tail % LOCAL_QUEUE_SIZE] = task;
    atomicStore(&q->tail, tail + 1);
    return true;
}

bool TryDequeueLocal(LocalQueue* q, out TaskDesc task) {
    uint head = atomicLoad(&q->head);
    uint tail = atomicLoad(&q->tail);
    
    if (head >= tail) {
        return false;  // Queue empty
    }
    
    task = q->tasks[head % LOCAL_QUEUE_SIZE];
    atomicStore(&q->head, head + 1);
    return true;
}

3.2 Global Queue (MPMC)

The global queue uses fetch-and-add for lock-free MPMC:

bool TryEnqueueGlobal(TaskDesc task) {
    uint slot = atomicAdd(&g_GlobalQueue.tail, 1);
    
    // Check for overflow
    if (slot >= g_GlobalQueue.head + GLOBAL_QUEUE_SIZE) {
        atomicSub(&g_GlobalQueue.tail, 1);  // Rollback
        return false;
    }
    
    g_GlobalQueue.tasks[slot % GLOBAL_QUEUE_SIZE] = task;
    atomicOr(&g_SlotValid[slot % GLOBAL_QUEUE_SIZE], 1);
    return true;
}

bool TryDequeueGlobal(out TaskDesc task) {
    while (true) {
        uint head = atomicLoad(&g_GlobalQueue.head);
        uint tail = atomicLoad(&g_GlobalQueue.tail);
        
        if (head >= tail) {
            return false;
        }
        
        // Wait for slot to be valid
        if (!atomicLoad(&g_SlotValid[head % GLOBAL_QUEUE_SIZE])) {
            continue;  // Enqueue in progress
        }
        
        if (atomicCompareExchange(&g_GlobalQueue.head, head, head + 1) == head) {
            task = g_GlobalQueue.tasks[head % GLOBAL_QUEUE_SIZE];
            atomicAnd(&g_SlotValid[head % GLOBAL_QUEUE_SIZE], 0);
            return true;
        }
    }
}

3.3 Work Stealing

When local and global queues are empty, workers steal from other SMs:

bool TrySteal(uint mySM, out TaskDesc task) {
    // Round-robin through other SMs
    uint victim = (mySM + g_StealAttempt++) % NUM_SMS;
    
    if (victim == mySM) {
        victim = (victim + 1) % NUM_SMS;
    }
    
    LocalQueue* victimQueue = &g_LocalQueues[victim];
    
    // Steal from tail (opposite end from victim's consumption)
    uint tail = atomicLoad(&victimQueue->tail);
    uint head = atomicLoad(&victimQueue->head);
    
    if (tail - head <= 1) {
        return false;  // Not enough to steal
    }
    
    uint stealIdx = tail - 1;
    if (atomicCompareExchange(&victimQueue->tail, tail, stealIdx) == tail) {
        task = victimQueue->tasks[stealIdx % LOCAL_QUEUE_SIZE];
        return true;
    }
    
    return false;
}

4. Termination Detection

4.1 The Challenge

Detecting when all work is complete without global synchronization is non-trivial. Workers cannot simply check queue emptiness—work may be in-flight (dequeued but not yet completed) or about to be enqueued.

4.2 Distributed Termination Protocol

We adapt the Dijkstra-Scholten termination algorithm:

1. Maintain global counter of in-flight tasks
2. Increment on dequeue, decrement on completion
3. Terminate when counter reaches zero AND all queues empty

__device__ uint g_InFlightCount;
__device__ uint g_TerminationFlag;

void ExecuteTask(TaskDesc task) {
    g_FunctionTable[task.functionId](...);
    
    uint remaining = atomicSub(&g_InFlightCount, 1) - 1;
    
    if (remaining == 0) {
        // Check all queues truly empty
        if (AllQueuesEmpty()) {
            atomicStore(&g_TerminationFlag, 1);
        }
    }
}

bool ShouldTerminate() {
    return atomicLoad(&g_TerminationFlag) == 1;
}

4.3 Handling New Work After Termination Check

If CPU enqueues work after termination flag is set:

void CPUEnqueue(TaskDesc task) {
    // Reset termination flag
    atomicStore(&g_TerminationFlag, 0);
    
    // Increment in-flight (enqueue counts as in-flight until dequeued)
    atomicAdd(&g_InFlightCount, 1);
    
    EnqueueToGlobal(task);
    
    // Workers may have exited - relaunch if needed
    if (atomicLoad(&g_ActiveWorkers) == 0) {
        ReLaunchWorkers();
    }
}

5. Integration Patterns

5.1 Adaptive Mesh Refinement

Tasks spawn child tasks based on error metric:

void RefineMeshTask(uint dataOffset, uint dataSize) {
    MeshPatch patch = LoadPatch(dataOffset);
    float error = ComputeError(patch);
    
    if (error > threshold) {
        // Subdivide and enqueue children
        MeshPatch children[4];
        Subdivide(patch, children);
        
        for (int i = 0; i < 4; i++) {
            TaskDesc childTask = CreateTask(REFINE_MESH, StorePatch(children[i]));
            Enqueue(childTask);
        }
    } else {
        // Emit final patch
        OutputPatch(patch);
    }
}

5.2 Progressive Ray Tracing

Trace rays until quality threshold:

void TraceRayTask(uint dataOffset, uint dataSize) {
    RayState ray = LoadRay(dataOffset);
    
    TraceRay(ray);
    AccumulateSample(ray);
    
    float variance = ComputeVariance(ray.pixel);
    
    if (variance > varianceThreshold && ray.sampleCount < maxSamples) {
        // Continue tracing
        TaskDesc nextTask = CreateTask(TRACE_RAY, StoreRay(ray));
        Enqueue(nextTask);
    }
}

5.3 Deferred Shading Pipeline

Producer-consumer between stages:

void RasterizeTask(...) {
    // Produce fragments
    Fragment frags[16];
    Rasterize(..., frags);
    
    for each frag:
        TaskDesc shadeTask = CreateTask(SHADE_FRAGMENT, StoreFrag(frag));
        Enqueue(shadeTask);
}

void ShadeFragmentTask(...) {
    Fragment frag = LoadFrag(...);
    float4 color = Shade(frag);
    WritePixel(frag.coord, color);
}

6. Results

6.1 Microbenchmarks

Task throughput (tasks/second) on RTX 4090:

Task Size       | Dispatch/Task | PTP      | Speedup
----------------|---------------|----------|--------
1μs             | 50K           | 180K     | 3.6×
10μs            | 95K           | 195K     | 2.1×
100μs           | 99K           | 200K     | 2.0×
1ms             | 100K          | 200K     | 2.0×

PTP eliminates dispatch overhead, providing consistent 200K task/s regardless of task size.

6.2 Adaptive Mesh Refinement

Terrain with adaptive subdivision:

Method          | Time (ms) | Triangles | Quality
----------------|-----------|-----------|--------
Fixed 6 levels  | 4.2       | 262K      | Over-refined
Dispatch/level  | 8.1       | 68K       | Optimal
PTP adaptive    | 2.5       | 68K       | Optimal

PTP is 3.2× faster than dispatch-per-level while achieving optimal refinement.

6.3 Progressive Ray Tracing

Converge to 0.01 variance:

Method              | Time (ms) | Samples/Pixel
--------------------|-----------|---------------
Fixed 64 spp        | 12.4      | 64 (uniform)
Dispatch/iteration  | 18.2      | 31 (adaptive)
PTP adaptive        | 8.6       | 31 (adaptive)

PTP enables efficient adaptive sampling with 2.1× speedup over iterative dispatch.

6.4 Memory Overhead

Queue memory: 8MB (supports 1M concurrent tasks)
Task descriptors: 16 bytes each
Function table: 1KB (64 functions)

Total overhead: <10MB, negligible for GPU workloads.

7. Conclusion

Persistent Thread Pools fundamentally change GPU compute programming from dispatch-centric to task-centric. By maintaining long-running worker threads that pull from global queues, PTP eliminates dispatch overhead and enables dynamic work patterns.

The lock-free queue hierarchy—local per-SM queues with global overflow and work stealing—balances locality with load balancing. Distributed termination detection provides correct completion signaling without global barriers.

PTP enables algorithms previously impractical on GPUs: adaptive refinement, progressive rendering, and producer-consumer pipelines all become efficient. We believe PTP represents the future of GPU compute, moving from static parallelism to dynamic, responsive execution.
