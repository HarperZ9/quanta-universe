Neural Radiance Transfer: Compact Neural Networks for Real-Time Precomputed Radiance Transfer

Harper Research Division

Harper Engine Technical Whitepaper HE-WP-008

Abstract

We present Neural Radiance Transfer (NRT), replacing traditional PRT matrices with tiny neural networks that encode object-specific light transport. The networks enable real-time SH rotation and evaluation while reducing storage from megabytes to kilobytes per object.

1. Introduction

Precomputed Radiance Transfer (PRT) enables real-time global illumination by precomputing how objects respond to incident lighting. The transfer function—mapping incident spherical harmonics to exitant radiance—is stored as a dense matrix per vertex. For order-3 SH (9 coefficients) with RGB output, this requires 27×9=243 floats (972 bytes) per vertex.

For detailed meshes with millions of vertices, PRT storage becomes prohibitive. Compression techniques (clustered PCA, wavelet encoding) reduce size but sacrifice quality or require complex decompression.

We propose Neural Radiance Transfer (NRT), encoding the transfer function as a compact neural network. A single network per object (or object class) learns the mapping from incident SH, view direction, and surface parameters to exitant radiance. Storage reduces from megabytes to 2-16KB per object while enabling smooth LOD transitions impossible with discrete per-vertex data.

2. Method

2.1 Network Architecture

Input (16D):
- Incident SH coefficients: 9 floats (RGB combined via luminance weighting)
- View direction: 3 floats (world space)
- Surface parameters: 4 floats (uv, normal deviation, curvature)

Hidden layers:
- 2 layers × 32 neurons
- ReLU activation

Output (3D):
- Exitant radiance: RGB

Total parameters: (16×32 + 32) + (32×32 + 32) + (32×3 + 3) = 1,731 floats = 6.9KB

2.2 Training Data Generation

For each object, we generate training data via offline rendering:

1. Sample N surface points (10K-100K depending on complexity)
2. For each point, sample M incident SH configurations (1K random + key lighting scenarios)
3. For each configuration, render ground-truth exitant radiance via path tracing
4. Training set: N × M × (input, output) pairs

Training converges in ~5 minutes on consumer GPU.

2.3 View-Dependent Effects

The view direction input enables glossy transfer effects. The network learns to interpolate between diffuse-dominated response (view away from reflection) and specular-dominated response (view aligned with reflection direction).

For highly specular materials, we increase network capacity (64 neurons) to capture sharper angular variation.

2.4 Generalization Across Instances

For objects with shared materials (e.g., all wooden chairs), a single network encodes the entire class. Instance-specific variation is captured via:
- UV coordinates (texture-based albedo variation)
- Normal deviation (geometric detail difference)
- Runtime scale/rotation (handled via SH rotation of input)

3. Runtime Evaluation

3.1 SH Rotation

Incident lighting in world space must be rotated to object-local space for network input. We use ZYZ Euler angle decomposition of the object's rotation matrix, applying the corresponding SH rotation matrices.

For order-2 SH, rotation requires 9×9 matrix multiplication (81 operations). This is comparable to the network's 32×16=512 multiply-adds, so rotation overhead is modest.

3.2 GPU Implementation

Network weights are stored in constant memory (6.9KB easily fits). Evaluation is fully parallelized:

float3 evaluate_nrt(float[9] incident_sh, float3 view_dir, float4 surface_params) {
    // Layer 1
    float hidden1[32];
    for (int i = 0; i < 32; i++) {
        hidden1[i] = bias1[i];
        for (int j = 0; j < 16; j++)
            hidden1[i] += weight1[i][j] * input[j];
        hidden1[i] = max(0, hidden1[i]); // ReLU
    }
    // Layer 2 and output similar...
}

3.3 Batched Evaluation

For meshes with many vertices, we batch evaluations in groups of 64 for SIMD efficiency. Total cost: ~0.1ms per 100K vertices.

4. Results

4.1 Storage Comparison

Method                    | Storage (1M vertex mesh)
--------------------------|-------------------------
Raw PRT matrices          | 972 MB
Clustered PCA (16 clusters)| 48 MB
Wavelet compression       | 24 MB
NRT (ours)               | 6.9 KB

NRT achieves 140,000× compression versus raw storage.

---------------|---------|---------------|------
Diffuse statue | 0.012   | 0.019        | 0.015

NRT quality between raw and compressed traditional PRT.

4.3 Performance

At 1080p with 500K visible PRT vertices:
- Raw PRT lookup: 0.4ms
- Clustered PCA: 0.6ms
- NRT evaluation: 0.5ms

Comparable performance with vastly reduced memory.

5. Conclusion

Neural Radiance Transfer demonstrates that learned representations can replace explicit PRT storage with dramatic compression ratios. The approach enables PRT on memory-limited platforms (mobile, VR) and for complex scenes where traditional PRT storage is prohibitive.

The neural formulation also enables capabilities impossible with discrete storage: continuous LOD via network interpolation, smooth animation of transfer properties, and efficient handling of object classes via shared networks.

Future work includes extension to all-frequency transfer (beyond SH), integration with neural LOD systems, and learned compression of network weights for further size reduction.
