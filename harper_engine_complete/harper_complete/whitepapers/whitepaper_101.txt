Neural Geometry Compression: Learned Latent Space Representations for Real-Time Mesh Streaming

Harper Research Division

Advanced Rendering Technology Group

Technical Whitepaper HE-WP-101

Abstract

We present Neural Geometry Compression (NGC), achieving 20:1 mesh compression through learned latent space encoding. A variational autoencoder maps vertex positions to 128-dimensional latent vectors; runtime decoding via 3-layer MLP reconstructs geometry with 0.1mm accuracy. GPU-accelerated inference enables real-time decompression at 500K vertices/ms, enabling streaming of massive environments within bandwidth constraints previously limiting open-world games.

1. Introduction

Mesh data dominates storage in modern games—a single character may require 50MB of vertex data across LOD levels. Streaming this data from disk or network creates hard limits on environment scale and visual fidelity.

Traditional compression (quantization, delta encoding, Draco) achieves 4-8× reduction but struggles beyond this due to inherent mesh irregularity. We observe that mesh surfaces occupy a low-dimensional manifold within the space of all possible vertex configurations—capturing this manifold enables dramatic compression.

Neural Geometry Compression learns this manifold via variational autoencoder (VAE). The encoder maps mesh patches to compact latent vectors; the decoder reconstructs geometry from these vectors. Training on diverse mesh datasets teaches the network to represent common surface patterns efficiently.

Key contributions:
1. VAE architecture optimized for mesh patch encoding
2. Hierarchical latent organization matching LOD requirements
3. GPU inference pipeline achieving real-time decompression
4. Integration with existing mesh streaming systems

2. Network Architecture

2.1 Encoder Design

The encoder processes fixed-size mesh patches (64 vertices each):

Input: Vertex positions (64 × 3 floats) + connectivity (variable triangles)
Hidden: 256 → 128 → 128 neurons, ReLU activation
Output: μ (128 floats) + log(σ²) (128 floats) for VAE sampling

Connectivity is encoded implicitly through position ordering—vertices are sorted by principal component analysis, making connectivity learnable from positions.

2.2 Decoder Design

The decoder reconstructs from latent vectors:

Input: z (128 floats sampled from N(μ, σ²))
Hidden: 128 → 256 → 512 neurons, ReLU activation
Output: 64 × 3 floats (reconstructed positions)

Skip connections from encoder to decoder preserve high-frequency detail.

2.3 Loss Function

Total loss combines reconstruction and KL divergence:

L = λ_recon × ||x - x̂||² + λ_KL × KL(q(z|x) || p(z)) + λ_normal × ||n - n̂||²

where n represents vertex normals (computed from reconstructed positions). The normal term ensures surface orientation accuracy critical for shading.

3. Training Procedure

3.1 Dataset Construction

Training requires diverse mesh patches:
- 50K patches from scanned objects (photogrammetry)
- 50K patches from CAD models (hard surfaces)
- 50K patches from organic sculpts (characters)
- 50K patches from terrain (landscapes)

Patches are extracted via random sampling on mesh surfaces, normalized to unit cube.

3.2 Data Augmentation

Random transformations increase effective dataset size:
- Rotation: Random 3D rotation
- Scale: 0.8-1.2× uniform scaling
- Noise: Gaussian noise (σ = 0.01) on positions
- Decimation: Random edge collapse (80-100% vertices)

3.3 Training Schedule

Batch size: 256 patches
Optimizer: Adam (lr = 1e-4, β₁ = 0.9, β₂ = 0.999)
Schedule: Cosine annealing over 100 epochs
λ_recon = 1.0, λ_KL = 0.001, λ_normal = 0.5

KL weight is annealed from 0 to 0.001 over first 10 epochs to prevent posterior collapse.

4. GPU Inference Pipeline

4.1 Compute Shader Implementation

Decompression runs entirely on GPU:

[numthreads(64, 1, 1)]
void DecompressPatch(uint tid : SV_DispatchThreadID, uint gid : SV_GroupID) {
    // Load latent vector for this patch
    float latent[128];
    LoadLatent(gid, latent);
    
    // Decoder forward pass
    float h1[256] = Dense(latent, g_W1, g_B1, 128, 256);
    ReLU(h1, 256);
    
    float h2[512] = Dense(h1, g_W2, g_B2, 256, 512);
    ReLU(h2, 512);
    
    float output[192] = Dense(h2, g_W3, g_B3, 512, 192);
    
    // Write decompressed vertex
    uint vertexId = gid * 64 + tid;
    g_Vertices[vertexId] = float3(output[tid*3], output[tid*3+1], output[tid*3+2]);
}

4.2 Memory Layout

Latent vectors: 128 × 4 bytes = 512 bytes per patch
Weights: 128×256 + 256×512 + 512×192 ≈ 360KB total
Decompressed output: Written directly to vertex buffer

4.3 Streaming Integration

Latent vectors stream from disk/network → staging buffer → GPU buffer.
Decompression dispatched when vertices needed for rendering.
Results persist in GPU memory until evicted by LRU cache.

5. Results

5.1 Compression Ratio

Mesh Type        | Original | NGC     | Ratio
-----------------|----------|---------|-------
Character (50K)  | 600KB    | 32KB    | 18.7×
Vehicle (100K)   | 1.2MB    | 58KB    | 21.2×
Environment (1M) | 12MB     | 580KB   | 20.7×
Terrain (500K)   | 6MB      | 285KB   | 21.1×

Average: 20.4× compression.

5.2 Quality Metrics

Hausdorff distance (normalized to bounding box diagonal):
- Mean: 0.08mm
- 99th percentile: 0.24mm
- Max: 0.41mm

Visual quality: Indistinguishable from original at typical game camera distances.

5.3 Performance

Decompression throughput on RTX 4090:
- 512K vertices/ms (single batch)
- 8M vertices total budget @ 60fps

Memory bandwidth: 2.1 GB/s (latent read) vs 12.8 GB/s (uncompressed read) = 6× bandwidth reduction.

5.4 Comparison to Draco

Metric              | Draco    | NGC (Ours)
--------------------|----------|------------
Compression ratio   | 7.2×     | 20.4×
Decode time (100K)  | 4.2ms    | 0.2ms
GPU-native decode   | No       | Yes

6. Conclusion

Neural Geometry Compression demonstrates that learned representations dramatically outperform traditional compression for mesh data. The 20× compression ratio enables streaming of environments 20× larger within fixed bandwidth, or equivalently, 20× higher geometric detail at current bandwidth.

GPU-native decompression eliminates CPU bottlenecks and integrates seamlessly with modern rendering pipelines. The technique is particularly valuable for:
- Open-world streaming with limited storage
- Network-delivered content (cloud gaming, AR/VR)
- Mobile platforms with severe memory constraints

Future work will extend to animated meshes (temporal coherence in latent space) and learned connectivity compression.
